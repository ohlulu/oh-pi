# Development Search Strategy

## When to Use
Query mentions a specific framework, library, API, error message, programming language, or debugging scenario.

## Target Sites
- Official documentation sites (e.g., developer.apple.com, docs.python.org, react.dev)
- github.com — issues (open + closed), discussions, release notes, README
- stackoverflow.com / stackexchange.com — Q&A with voted answers
- Developer blogs from framework maintainers
- WWDC transcripts / session notes (Apple ecosystem)

## Query Patterns
- "{topic} site:github.com issues"
- "{topic} site:stackoverflow.com"
- "{framework} {topic} official documentation"
- "{error message}" (exact match in quotes)
- "{topic} release notes changelog 2025"
- "{topic} migration guide"

## Source Evaluation
- **Prefer**: official docs, release notes, maintainer blog posts, highly-voted SO answers (50+ votes)
- **Deprioritize**: outdated tutorials (pre-2024), beginner-level how-tos, AI-generated code explanations
- **Red flags**: answers with 0 votes or negative votes, tutorials for wrong version, copy-paste content farms
